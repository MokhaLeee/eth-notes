PTPDv2 2.3.1

usage: ptpd2 <options> < --section:key=value...>

WARNING: Any command-line options take priority over options from config file!

Basic options: 

-c --config-file [path] 	Configuration file
-k --check-config		Check configuration and exit
-v --version			Print version string
-h --help			Show this help screen
-H --long-help			Show detailed help for all settings and behaviours
-e --explain [section:key]	Show help for a single setting
-O --default-config		Output default configuration (usable as config file)
-L --ignore-lock		Skip lock file checks and locking
-A --auto-lockfile		Use preset / port mode specific lock files
-l --lockfile [path]		Specify path to lock file
-p --print-lockfile		Print path to lock file and exit (useful for init scripts)
-R --lock-directory [path]	Directory to store lock files
-f --log-file [path]		global:log_file=[path]		Log file
-S --statistics-file [path]	global:statistics_file=[path]	Statistics file
-T --dot2as			Enable 802.1AS support. This has been deprecated in favour of ptpengine:profile.
-N --noswitch			Enable no switch mode
-K --resmgr			Enable the ptpd2 resmgr
-F --fp ClockId:PortId		Force Master Clock Identity and ignore announce packets
				or ignore BMCA and immediately act as a master
-X --correction-step [n]	Correction (ns) to add to offset when stepping the clock
-J --max-step [n]		Maximum offset in ns before stepping the clock
-I --reqannounceinterval	Request announce interval of -1 in slave mode
-Q --sw-timestamps-only	Don't try to enable HW timestamping and just use SW timestamps
-q --disable-signaling		Do not handle signaling messages
-o --servo-optimization	Enable experimental servo optimizations. HW support required.
-x --extra-opts STRING[,STRING]  Clock update configuration using suboptions:
					noSysClkAdj         Disable adjustments to the system clock.
					noPTPClkAdj         Disable adjustments to the PTP clock with supported hardware.
					noSetPTPTimeFromSys Disable initialization of the PTP clock from the system clock with supported hardware.
                                 Configuration file for additional parameters:					<FILENAME>          The path and filename to an .ini file.

Basic PTP protocol and daemon configuration options: 

Command-line option		Config key(s)			Description
------------------------------------------------------------------------------------
-i --interface [dev]		ptpengine:interface=<dev>	Interface to use (required)
-d --domain [n] 		ptpengine:domain=<n>		PTP domain number
-s --slaveonly	 	 	ptpengine:preset=slaveonly	Slave only mode
-m --masterslave 		ptpengine:preset=masterslave	Master, slave when not best GM
-M --masteronly 		ptpengine:preset=masteronly	Master, passive when not best GM
-y --hybrid			ptpengine:ip_mode=hybrid	Hybrid mode (multicast for sync
								and announce, unicast for delay
								request and response)
-U --unicast			ptpengine:ip_mode=unicast	Unicast mode
-g --unicast-negotiation	ptpengine:unicast_negotiation=y Enable unicast negotiation (signaling)
-u --unicast-destinations 	ptpengine:ip_mode=unicast	Unicast destination list
     [ip/host, ...]		ptpengine:unicast_destinations=<ip/host, ...>

-E --e2e			ptpengine:delay_mechanism=E2E	End to end delay detection
-P --p2p			ptpengine:delay_mechanism=P2P	Peer to peer delay detection

-a --delay-override 		ptpengine:log_delayreq_override Override delay request interval
                                                               announced by master
-r --delay-interval [n] 	ptpengine:log_delayreq_interval=<n>	Delay request interval
									(log 2)

-n --noadjust			clock:no_adjust			Do not adjust the clock
-D<DD...> --debug		global:debug_level=<level>	Debug level
-C --foreground			global:foreground=<Y/N>		Don't run in background
-V --verbose			global:verbose_foreground=<Y/N>	Run in foreground, log all
								messages to standard output

------------------------------------------------------------------------------------

ptpd2 accepts a configuration file (.ini style) where settings are either
grouped into sections:

[section]
; comment
key = value

or are specified as:

section:key = value

All settings can also be supplied as command-line parameters (after other options):

--section:key=<value> --section:key <value> -section:key=<value> -section:key <value>

To see the full help for all configurable options, run: ptpd2 with -H or --long-help
To see help for a single setting only, run: ptpd2 with -e (--explain) [section:key]
To see all default settings with descriptions, run: ptpd2 with -O (--default-config)

------------------------------------------------------------------------------------

PTPDv2 compatibility options for migration from versions below 2.3.0:

-b [dev]			Network interface to use
-i [n]				PTP domain number
-G				'Master mode with NTP' (ptpengine:preset=masteronly)
-W				'Master mode without NTP' (ptpengine:preset=masterslave) 
-Y [n]				Delay request interval (log 2)
-t 				Do not adjust the clock

Note 1: the above parameters are deprecated and their use will issue a warning.
Note 2: -U and -g options have been re-purposed in 2.3.1
.


============== Full list of ptpd2 settings ========

setting: ptpengine:interface (--ptpengine:interface)
   type: STRING
  usage: Network interface to use - eth0, igb0 etc. (required).
default: [none]

setting: ptpengine:backup_interface (--ptpengine:backup_interface)
   type: STRING
  usage: Backup network interface to use - eth0, igb0 etc. When no GM available, 
	 slave will keep alternating between primary and secondary until a GM is found.

default: [none]

setting: ptpengine:preset (--ptpengine:preset)
   type: SELECT
  usage: PTP engine preset:
	 none	     = Defaults, no clock class restrictions
        masteronly  = Master, passive when not best master (clock class 0..127)
        masterslave = Full IEEE 1588 implementation:
                      Master, slave when not best master
	 	       (clock class 128..254)
        slaveonly   = Slave only (clock class 255 only)

options: none masteronly masterslave slaveonly 
default: slaveonly

setting: ptpengine:transport (--ptpengine:transport)
   type: SELECT
  usage: Transport type for PTP packets. Ethernet transport requires libpcap support.
options: ipv4 ethernet 
default: ipv4

setting: ptpengine:profile (--ptpengine:profile)
   type: SELECT
  usage: PTP profile:
      PTP = IEEE 1588 default profile
     gPTP = 802.1AS PTP profile
  AUTOSAR = AUTOSAR PTP profile
options: PTP gPTP AUTOSAR 
default: PTP

setting: ptpengine:ip_mode (--ptpengine:ip_mode)
   type: SELECT
  usage: IP transmission mode (requires IP transport) - hybrid mode uses
	 multicast for sync and announce, and unicast for delay request and
	 response; unicast mode uses unicast for all transmission.
	 When unicast mode is selected, destination IP(s) may need to be configured
	(ptpengine:unicast_destinations).
options: multicast unicast hybrid 
default: multicast

setting: ptpengine:unicast_negotiation (--ptpengine:unicast_negotiation)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Enable unicast negotiation support using signaling messages

default: N

setting: ptpengine:disable_bmca (--ptpengine:disable_bmca)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Disable Best Master Clock Algorithm for unicast masters:
        Only effective for masteronly preset - all Announce messages
        will be ignored and clock will transition directly into MASTER state.

default: N

setting: ptpengine:unicast_negotiation_listening (--ptpengine:unicast_negotiation_listening)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: When unicast negotiation enabled on a master clock, 
	 reply to transmission requests also in LISTENING state.
default: N

setting: ptpengine:use_libpcap (--ptpengine:use_libpcap)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Use libpcap for sending and receiving traffic (automatically enabled
	 in Ethernet mode).
default: N

setting: ptpengine:delay_mechanism (--ptpengine:delay_mechanism)
   type: SELECT
  usage: Delay detection mode used - use DELAY_DISABLED for syntonisation only
	 (no full synchronisation).
options: E2E P2P DELAY_DISABLED 
default: E2E

setting: ptpengine:domain (--ptpengine:domain)
   type: INT (min: 0, max: 127)
  usage: PTP domain number.
default: 0

setting: ptpengine:port_number (--ptpengine:port_number)
   type: INT (min: 1, max: 65534)
  usage: PTP port number (part of PTP Port Identity - not UDP port).
        For ordinary clocks (single port), the default should be used, 
        but when running multiple instances to simulate a boundary clock, 
        The port number can be changed.
default: 1

setting: ptpengine:any_domain (--ptpengine:any_domain)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Usability extension: if enabled, a slave-only clock will accept
	 masters from any domain, while preferring the configured domain,
	 and preferring lower domain number.
	 NOTE: this behaviour is not part of the standard.
default: N

setting: ptpengine:slave_only (--ptpengine:slave_only)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Slave only mode (sets clock class to 255, overriding value from preset).
default: Y

setting: ptpengine:inbound_latency (--ptpengine:inbound_latency)
   type: INT
  usage: Specify latency correction (nanoseconds) for incoming packets.
default: 0

setting: ptpengine:outbound_latency (--ptpengine:outbound_latency)
   type: INT
  usage: Specify latency correction (nanoseconds) for outgoing packets.
default: 0

setting: ptpengine:offset_shift (--ptpengine:offset_shift)
   type: INT
  usage: Apply an arbitrary shift (nanoseconds) to offset from master when
	 in slave state. Value can be positive or negative - useful for
	 correcting for antenna latencies, delay assymetry
	 and IP stack latencies. This will not be visible in the offset 
	 from master value - only in the resulting clock correction.
default: 0

setting: ptpengine:always_respect_utc_offset (--ptpengine:always_respect_utc_offset)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Compatibility option: In slave state, always respect UTC offset
	 announced by best master, even if the the
	 currrentUtcOffsetValid flag is announced FALSE.
	 NOTE: this behaviour is not part of the standard.
default: N

setting: ptpengine:prefer_utc_offset_valid (--ptpengine:prefer_utc_offset_valid)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Compatibility extension to BMC algorithm: when enabled,
	 BMC for both master and save clocks will prefer masters
	 nannouncing currrentUtcOffsetValid as TRUE.
	 NOTE: this behaviour is not part of the standard.
default: N

setting: ptpengine:require_utc_offset_valid (--ptpengine:require_utc_offset_valid)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)
  usage: Compatibility option: when enabled, ptpd2 will ignore
	 Announce messages from masters announcing currentUtcOffsetValid
	 as FALSE.
	 NOTE: this behaviour is not part of the standard.
default: N

setting: ptpengine:unicast_grant_duration (--ptpengine:unicast_grant_duration)
   type: INT (min: 30, max: 604800)
  usage: Time (seconds) unicast messages are requested for by slaves
	 when using unicast negotiation, and maximum time unicast message
	 transmission is granted to slaves by masters

default: 300

setting: ptpengine:log_announce_interval (--ptpengine:log_announce_interval)
   type: INT (min: -4, max: 7)
  usage: PTP announce message interval in master state. When using unicast negotiation, for
	 slaves this is the minimum interval requested, and for masters
	 this is the only interval granted.
	(expressed as log 2 i.e. -1=0.5s, 0=1s, 1=2s etc.)
default: 1

setting: ptpengine:log_announce_interval_max (--ptpengine:log_announce_interval_max)
   type: INT (min: -1, max: 7)
  usage: Maximum Announce message interval requested by slaves when using unicast negotiation,
	(expressed as log 2 i.e. -1=0.5s, 0=1s, 1=2s etc.)
default: 5

setting: ptpengine:announce_receipt_timeout (--ptpengine:announce_receipt_timeout)
   type: INT (min: 2, max: 255)
  usage: PTP announce receipt timeout announced in master state.
default: 6

setting: ptpengine:announce_receipt_grace_period (--ptpengine:announce_receipt_grace_period)
   type: INT (min: 0, max: 20)
  usage: PTP announce receipt timeout grace period in slave state:
	 when announce receipt timeout occurs, disqualify current best GM,
	 then wait n times announce receipt timeout before resetting.
	 Allows for a seamless GM failover when standby GMs are slow
	 to react. When set to 0, this option is not used.
default: 0

setting: ptpengine:log_sync_interval (--ptpengine:log_sync_interval)
   type: INT (min: -7, max: 7)
  usage: PTP sync message interval in master state. When using unicast negotiation, for

	(expressed as log 2 i.e. -1=0.5s, 0=1s, 1=2s etc.)

setting: ptpengine:log_sync_interval_max (--ptpengine:log_sync_interval_max)
   type: INT (min: -1, max: 7)
  usage: Maximum Sync message interval requested by slaves when using unicast negotiation,

default: 5

setting: ptpengine:log_delayreq_override (--ptpengine:log_delayreq_override)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)

setting: ptpengine:log_delayreq_auto (--ptpengine:log_delayreq_auto)
   type: BOOLEAN (value must start with t/T/y/Y/1/f/F/n/N/0)

setting: ptpengine:log_delayreq_interval_initial (--ptpengine:log_delayreq_interval_initial)

Possible internal states:
  init:        INITIALIZING
  flt:         FAULTY
  lstn_init:   LISTENING (first time)
  lstn_reset:  LISTENING (non first time)
  pass:        PASSIVE Master (not best in BMC, not announcing)
  uncl:        UNCALIBRATED
  slv:         SLAVE
  pmst:        PRE Master
  mst:         ACTIVE Master
  dsbl:        DISABLED
  ?:           UNKNOWN state

Handled signals:
  SIGHUP         Reload configuration file and close / re-open log files
  SIGUSR1        Manually step clock to current OFM value
                 (overides clock:no_reset, but honors clock:no_adjust)
  SIGUSR2	  Dump all PTP protocol counters to current log target
                 (and clear if ptpengine:sigusr2_clears_counters set)

  SIGINT|SIGTERM Close open files, remove lock file and exit cleanly
  SIGKILL        Force an unclean exit

BMC Algorithm defaults:
  Software:   P1(128) > Class(13|248) > Accuracy("unk"/0xFE)   > Variance(65536) > P2(128)
